# 100 упражнений для совершенствования в GOLANG

## Введение

Я не нашёл хорошего репозитория со 100 упражнениями для языка программирования Go, и поскольку это прекрасный язык — решил создать его самостоятельно :smile: Не стесняйтесь вносить вклад и использовать его!

Представленные упражнения предполагают достаточно базовый уровень объектно-ориентированного программирования, однако их можно выполнять и без такого опыта.

## Обзор

Основная идея состоит в том, что упражнения структурированы следующим образом:

- С 1 по 15   : Базовые упражнения
- С 16 по 19  : Работа с файлами
- С 20 по 31  : Упражнения среднего уровня
- С 32 по 44  : Конкурентность
- С 45 по 48  : Время
- С 49 по 53  : JSON
- С 54 по 61  : Контекст
- С 62 по 68  : Интерфейсы
- С 69 по 82  : HTTP и API
- С 83 по 89  : Тестирование и разное
- С 90 по 100 : База данных (нереляционная (mongo)) + CRUD API с Fiber

Каждая папка содержит одно упражнение и одно решение. Существует множество способов достичь одного и того же результата, поэтому любой подход засчитывается.

```bash
├── n-ое_упражнение     # <- номер упражнения
│    ├── README.md      # <- описание упражнения и информация + подсказки (для книги!)
│    ├── main.go
│    ├── [.env]
│    └── solution       # <- папка с решением
│       └── solution.go
├── go.mod
└── go.sum
```

## Предварительные требования

### Git

Git — это распределённая система контроля версий, которая отслеживает изменения файлов.
Она часто используется для управления исходным кодом программистами, разрабатывающими программное обеспечение совместно; например, более 1000 человек могут одновременно работать над одним проектом, не мешая друг другу.

Если вы никогда не слышали о git и интересуетесь программированием, это инструмент, знание которого для вас крайне важно.

Текст этой книги хранится в git-репозитории, и в идеале вы должны уметь сделать форк репозитория, чтобы запускать код локально и экспериментировать.

[Скачать git](https://git-scm.com/downloads)

### Github / Системы контроля версий

Github — это онлайн-платформа для хранения кода. Код также можно хранить локально (без git) или на любой другой платформе (Gitlab, Bitbucket, Azure, AWS CodeCommit и т. д.).

При написании кода настоятельно рекомендуется использовать систему контроля версий, чтобы иметь сохранённые состояния, в которых код работает как ожидается, наподобие контрольных точек.

Лично мне нравится GitHub благодаря широкому сообществу, удобному пользовательскому опыту, а также тому, что некоторые из наиболее известных репозиториев с открытым исходным кодом хранятся там: [linux](https://github.com/torvalds/linux), [bitcoin](https://github.com/bitcoin/bitcoin), [kubernetes](https://github.com/kubernetes/kubernetes) и даже сам язык [go](https://github.com/golang/go) в виде зеркала.

### IDE на ваш выбор

[IDE](https://en.wikipedia.org/wiki/Integrated_development_environment) — это то, где вы будете писать код.
IDE предоставляют широкий спектр функций и плагинов, помогающих писать код удобнее, быстрее, с меньшим количеством ошибок и эффективнее.

Visual Studio Code — отличная бесплатная IDE, если вы новичок в программировании. Но не стесняйтесь использовать то, с чем вам удобно. (Код также можно писать в простом блокноте или текстовых редакторах)

[Скачать Visual Studio Code](https://code.visualstudio.com/)

### Golang

Golang — это язык программирования, разработанный в Google в 2007 году Робертом Гризмером, Робом Пайком и Кеном Томпсоном и публично объявленный в ноябре 2009 года.

Если и есть одна вещь, которую большинство людей знают о Go, так это то, что он создан для конкурентности. Ни одно введение в Go не обходится без демонстрации горутин и каналов.

Но когда люди слышат слово «конкурентность», они часто думают о параллелизме — связанном, но совершенно отличном понятии. В программировании конкурентность — это совместное выполнение независимо работающих процессов, а параллелизм — одновременное выполнение (возможно, связанных) вычислений. Конкурентность — о том, как справляться со многими вещами одновременно. Параллелизм — о том, как делать многие вещи одновременно.

#### Параллелизм и конкурентность

Простой пример конкурентности: представьте, что вам нужно съесть целый торт и спеть целую песню.
Правило: вы поёте и едите одновременно. Можно съесть весь торт, а потом спеть всю песню, или съесть половину торта, спеть половину песни, потом повторить, и т. д.

Следуя этому примеру, параллелизм означал бы, что друг, умеющий петь, присоединяется к вам в этом задании: каждый из вас получает свой процесс (например, друг поёт, а вы едите торт), и вы делаете это одновременно, параллельно.

Параллелизм в основе своей требует аппаратного обеспечения с несколькими процессорными блоками. На одноядерном процессоре можно получить конкурентность, но НЕ параллелизм. Параллелизм — это особый вид конкурентности, при котором задачи действительно выполняются одновременно.
